# 구현 출력 계약 (implementation)

코드 변경을 제안하거나 실행했을 때의 출력 형식.

## 변경 요약
1~3개 글머리. 무엇을 했고 왜 했는지.

## 영향 범위
| 파일 | 변경 유형 | 라인 수 (대략) |
|---|---|---|
| `src/foo.ts` | 신규 | +42 |
| `src/bar.ts` | 수정 | +12 / -8 |

## 변경 내역
각 파일·블록당:

- 파일: `src/foo.ts:23-45`
- 변경: (구체적 무엇)
- 이유: (왜 이렇게 했는가, 대안과 기각 사유)
- 코드:

```ts
// 핵심 변경 예시 (긴 diff는 생략하고 인터페이스/시그니처 위주)
```

## 검증 방법
- 실행한 테스트 / 명령어 / 결과.
- 사용자가 직접 검증할 수 있는 절차.

## 미반영 / 후속
- 의도적으로 미반영한 항목과 사유.
- 후속 작업으로 권고하는 것.

## 리스크
- 회귀 가능성·성능·보안 리스크.

## 확신도
`high` | `medium` | `low` — 한 줄 사유.
