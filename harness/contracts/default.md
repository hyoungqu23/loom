# 기본 출력 계약 (default)

다음 Markdown 구조를 정확히 사용하라. 각 섹션은 비어 있어도 헤더는 유지하고 본문에 `해당 없음`을 적는다.

## 요약
1~3개 글머리. 핵심 결론을 가장 먼저.

## 관찰
- 입력에서 직접 도출한 사실/관찰. 코드·문서 인용을 동반.

## 리스크
- 알려진 위험·미지의 영역·갭. 없으면 `해당 없음`.

## 결정
- 내려진 결정 또는 권고. 없으면 `해당 없음`.

## 다음 액션
- 실행 가능한 다음 단계 (담당이 누구인지 명확하게).

## 확신도
`high` | `medium` | `low` — 한 줄 사유.
